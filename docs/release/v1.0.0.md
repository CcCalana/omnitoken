# OmniToken v1.0.0

Released: 2026-05-23

OmniToken v1 是面向企业内部平台团队的自托管 AI 访问网关。本版本完成"底座三角"——性价比资源、权限/额度、安全/审计——的最小可上线形态。

## v1 提供什么

### 性价比资源：多 provider key 池

- **跨 provider 凭证池**：单网关同时管理 Volcano Ark 和 DeepSeek 两家上游 provider，AES-256-GCM 信封加密入库，gateway 启动后从 DB decrypt 入池。
- **跨 provider 自动 fallback**：当虚拟模型路由的目标 provider 全部 429/degraded 时，selector 自动切到其他 provider 的 credential 继续重试；usage 流水按实际命中的 provider+credential_id 归因。
- **多 key priority 轮询**：v1 按 priority 数字升序选择，429 触发 fallback；同一 priority 内按 weight 轮询。
- **虚拟模型路由**：客户端用稳定别名（`chat-fast` / `chat-balanced` / `chat-quality` / `chat-code` / `chat-experimental`），gateway 改写为真实 provider 模型字符串。当前默认映射：`chat-fast → DeepSeek deepseek-v4-flash`，其余仍走 Ark coding plan。

**实测**：30 并发 × 30 秒 × 900 请求 = 100.0% 成功率（DeepSeek 多 key），与之前单 Ark coding plan 17.1% 形成强对照。详见 `docs/release/v1-concurrency-rerun-2026-05-22.md`。

### 权限/额度

- **三角色 RBAC**：admin / member / viewer，硬编码 policy matrix。
- **月度 budget 强制**：每用户独立 `monthly_token_budget_limit`，usage 入账前 budget 校验，超额返 402 quota envelope。
- **admin 可编辑 quota**：admin 通过 web 改用户配额，PATCH 路径走 RBAC + audit。
- **虚拟密钥管理**：admin 端发 `omt_` 前缀虚拟密钥，明文仅生成时返回一次；DB 只存 `key_prefix` + bcrypt hash。

### 安全/审计

- **audit_logs 中间件**：admin 所有写操作（login / create_virtual_key / update_quota / create_credential / disable_credential 等）自动落审计日志，含 before/after snapshot。
- **异常 key 速率告警**：5 分钟窗口 RPM 超阈值（默认 60）的 virtual key 写 audit WARN。
- **日志安全基线**：网关日志默认不打 provider key / virtual key 明文 / prompt body / Authorization 头。流式响应统一错误信封。
- **会话登录 + bootstrap fallback**：admin 走 session/JWT 登录；`OMNITOKEN_ADMIN_BOOTSTRAP_TOKEN` 仅为本地 demo fallback，生产留空。

### 运维 UX

- **admin 网页加 key**：admin web 加 "Upstream Credentials" tab，运维填表新增/禁用 credential，无需 SSH 改 `.env`。
- **30 秒热加载**：gateway 每 30 秒（可配 `OMNITOKEN_CREDENTIAL_POLL_INTERVAL`）从 DB 拉一次凭证池并原子 swap，新增 key 自动生效无需重启。
- **OpenAI 兼容入口**：客户端用 `POST /v1/chat/completions` 与 `GET /v1/models`，与现有 OpenAI SDK 直接兼容。
- **Docker Compose 一键起**：`docker compose up` 启 gateway + admin + Postgres + Redis + NATS + 数据库迁移 + seed。

## 安装与升级

参考 `README.md` Quick Start。**v1 是首版**，无升级路径。

### 必备前置

- Docker（compose v2）
- 两组 secret：
  - `OMNITOKEN_MASTER_KEY`（64-hex 32-byte，admin 与 gateway 共享，用于信封加密）
  - 上游 provider keys（Ark coding plan key 或 DeepSeek API keys，写进 `.env`）
- DeepSeek 用户：到 `https://api.deepseek.com` 充值（最低 ¥10 即可，v1 测试规模总成本 < ¥1）

## Provider 支持

| Provider | 协议 | v1 状态 |
| --- | --- | --- |
| Volcano Ark coding plan | OpenAI-compatible | ✅ 实测多 key 池 + 429 fallback |
| DeepSeek 官方 API | OpenAI-compatible | ✅ 实测 30 并发 100% 成功率 |
| OpenAI / Azure OpenAI / Moonshot / 通义千问 / 其他 OpenAI 兼容 | OpenAI-compatible | ⏭ v1.1+ 加 provider whitelist 即可接入，协议层无需改 |
| Anthropic Messages | 独立协议 | ⏭ v1.1+ (T-020) |
| Google Gemini / Vertex / AWS Bedrock | 独立协议 | ⏭ v1.1+ |

## 已知限制

| 范围 | 说明 |
| --- | --- |
| 单实例 PG 连接上限 | mock 实测 50 并发不影响成功率；100+ 并发会撞 PG `too many clients`。v1 默认 RPS 上限 ≈ 30，更高并发是 v1.1 优化项（T-QUOTA-CACHE-PROBE）。 |
| 凭证 UPDATE 缺失 | admin UI 支持新增 + 禁用，**不支持编辑现有凭证**。需要改 priority / base_url 时走 disable + 新 add。 |
| 主密钥管理 | v1 通过 env 注入，admin 与 gateway 共享。KMS 集成是 v1.1+ 范围。 |
| 单 admin 实例 | v1 无 admin 多实例 / leader-election；polling 间无 jitter，多实例部署需先看 v1.1。 |
| polling 反应时间 | 默认 30 秒，可降到 1 秒级（DB 压力略增）。LISTEN/NOTIFY 即时通知是 v1.1。 |
| Anthropic / Gemini / Bedrock | 协议差异较大，留 v1.1+ 适配。当前 OpenAI-compatible 客户端可直接对接。 |

无。v1 是首个公开 release，不与历史 demo 版本兼容（旧 `.env` 单 key 路径仍保留为 fallback，但建议走 v1 多 provider 池）。
